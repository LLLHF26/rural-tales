## 三、表结构设计

### 3.1 users — 用户表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT UNSIGNED | PK, AUTO_INCREMENT | 用户 ID |
| phone | VARCHAR(11) | UNIQUE, NOT NULL | 手机号 |
| password_hash | VARCHAR(255) | NOT NULL | bcrypt 密码哈希 |
| nickname | VARCHAR(50) | DEFAULT '' | 昵称 |
| avatar | VARCHAR(500) | DEFAULT '' | 头像 URL |
| created_at | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 注册时间 |
| updated_at | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

**索引**：`UNIQUE uk_phone (phone)`

---

### 3.2 verification_codes — 验证码表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT UNSIGNED | PK, AUTO_INCREMENT | — |
| phone | VARCHAR(11) | NOT NULL | 手机号 |
| code | VARCHAR(10) | NOT NULL | 验证码 |
| expires_at | DATETIME | NOT NULL | 过期时间 |
| used | TINYINT(1) | DEFAULT 0 | 是否已使用 |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 发送时间 |

**索引**：`INDEX idx_phone_code (phone, code)`，`INDEX idx_expires (expires_at)`

---

### 3.3 villages — 乡村表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT UNSIGNED | PK, AUTO_INCREMENT | — |
| name | VARCHAR(100) | NOT NULL | 乡村名称 |
| description | TEXT | — | 乡村介绍 |
| cover_image | VARCHAR(500) | — | 封面图 URL |
| lat | DECIMAL(10,7) | NOT NULL | 纬度 |
| lng | DECIMAL(10,7) | NOT NULL | 经度 |
| address | VARCHAR(300) | — | 详细地址 |
| tags | JSON | — | 标签数组 `["古村落","非遗","茶园"]` |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | — |
| updated_at | DATETIME | DEFAULT CURRENT_TIMESTAMP ON UPDATE | — |

**索引**：`INDEX idx_tags ( (CAST(tags AS CHAR(100) ARRAY)) )` _（MySQL 8.0 多值索引，或使用 FULLTEXT）_

---

### 3.4 village_spots — 打卡点表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT UNSIGNED | PK, AUTO_INCREMENT | — |
| village_id | BIGINT UNSIGNED | FK → villages.id, NOT NULL | 所属乡村 |
| name | VARCHAR(100) | NOT NULL | 地点名称 |
| lat | DECIMAL(10,7) | NOT NULL | 纬度 |
| lng | DECIMAL(10,7) | NOT NULL | 经度 |
| description | TEXT | — | 地点介绍 |
| images | JSON | — | 图片 URL 数组 |
| sort_order | INT | DEFAULT 0 | 排序 |

**索引**：`INDEX idx_village (village_id)`

---

### 3.5 village_cultures — 文化条目表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT UNSIGNED | PK, AUTO_INCREMENT | — |
| village_id | BIGINT UNSIGNED | FK → villages.id, NOT NULL | 所属乡村 |
| type | ENUM('history','intangible','legend') | NOT NULL | 文化类型 |
| title | VARCHAR(200) | NOT NULL | 标题 |
| content | TEXT | — | 详细内容 |

**索引**：`INDEX idx_village_type (village_id, type)`

---

### 3.6 scripts — 剧本表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT UNSIGNED | PK, AUTO_INCREMENT | — |
| village_id | BIGINT UNSIGNED | FK → villages.id, NOT NULL | 所属乡村 |
| title | VARCHAR(200) | NOT NULL | 剧本标题 |
| cover_image | VARCHAR(500) | — | 封面图 URL |
| type | ENUM('mystery','history','family') | NOT NULL | 剧本类型 |
| difficulty | TINYINT UNSIGNED | NOT NULL, CHECK 1-5 | 难度等级 |
| estimated_duration | INT UNSIGNED | NOT NULL | 预计时长（分钟） |
| storyline | TEXT | — | 故事主线描述 |
| status | ENUM('draft','published','offline') | DEFAULT 'draft' | 状态 |
| rating_avg | DECIMAL(2,1) | DEFAULT 0 | 平均评分 |
| rating_count | INT UNSIGNED | DEFAULT 0 | 评分人数 |
| experience_count | INT UNSIGNED | DEFAULT 0 | 体验人次 |
| published_at | DATETIME | — | 发布时间 |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | — |
| updated_at | DATETIME | DEFAULT CURRENT_TIMESTAMP ON UPDATE | — |

**索引**：
- `INDEX idx_village_status (village_id, status)`
- `INDEX idx_type_status (type, status)`
- `INDEX idx_rating (rating_avg DESC)`
- `INDEX idx_experience (experience_count DESC)`

### 3.7 script_chapters — 剧本章节表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT UNSIGNED | PK, AUTO_INCREMENT | — |
| script_id | BIGINT UNSIGNED | FK → scripts.id, NOT NULL | 所属剧本 |
| title | VARCHAR(200) | NOT NULL | 章节标题 |
| sort_order | INT UNSIGNED | NOT NULL, DEFAULT 0 | 排序 |

**索引**：`INDEX idx_script_order (script_id, sort_order)`

---

### 3.8 script_nodes — 剧情节点表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT UNSIGNED | PK, AUTO_INCREMENT | — |
| script_id | BIGINT UNSIGNED | FK → scripts.id, NOT NULL | 所属剧本 |
| chapter_id | BIGINT UNSIGNED | FK → script_chapters.id, NOT NULL | 所属章节 |
| title | VARCHAR(200) | NOT NULL | 节点标题 |
| type | ENUM('dialogue','task_hub','ending') | NOT NULL | 节点类型 |
| scene_image | VARCHAR(500) | — | 场景图 URL |
| scene_audio | VARCHAR(500) | — | 场景背景音 URL |
| trigger_type | ENUM('gps','auto','manual') | DEFAULT 'auto' | 触发方式 |
| trigger_lat | DECIMAL(10,7) | — | 触发纬度 |
| trigger_lng | DECIMAL(10,7) | — | 触发经度 |
| trigger_radius | INT UNSIGNED | DEFAULT 30 | 触发半径（米） |
| dialogue_prompt | TEXT | — | AI 开场对话提示词 |
| npc_id | BIGINT UNSIGNED | FK → script_npcs.id | 关联的主要 NPC |
| config | JSON | NOT NULL | 节点配置（见下方结构） |
| sort_order | INT UNSIGNED | DEFAULT 0 | 排序 |

**config JSON 结构**：
```json
{
  "nextNodes": [2, 3],
  "hasBranch": true,
  "branchPrompt": "你选择跟老村长去祠堂，还是自己转转？",
  "branchOptions": [
    { "id": "opt_a", "label": "跟随老村长", "nextNodeId": 2 },
    { "id": "opt_b", "label": "独自探索", "nextNodeId": 3 }
  ]
}
```

**索引**：
- `INDEX idx_chapter (chapter_id)`
- `INDEX idx_script (script_id)`
- `INDEX idx_npc (npc_id)`

```json
[
```

```json
    "itemId": "item_001",
```

## 四、完整建表 SQL

```sql
-- ============================================================
-- 数据库初始化脚本
-- ============================================================

CREATE DATABASE IF NOT EXISTS rural_tourism
  DEFAULT CHARACTER SET utf8mb4
  DEFAULT COLLATE utf8mb4_unicode_ci;

USE rural_tourism;

-- -----------------------------------------------------------
-- 1. 用户表
-- -----------------------------------------------------------
CREATE TABLE users (
    id          BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
    phone       VARCHAR(11)     NOT NULL,
    password_hash VARCHAR(255)  NOT NULL,
    nickname    VARCHAR(50)     NOT NULL DEFAULT '',
    avatar      VARCHAR(500)    NOT NULL DEFAULT '',
    created_at  DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at  DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    PRIMARY KEY (id),
    UNIQUE KEY uk_phone (phone)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- -----------------------------------------------------------
-- 2. 验证码表
-- -----------------------------------------------------------
CREATE TABLE verification_codes (
    id          BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
    phone       VARCHAR(11)     NOT NULL,
    code        VARCHAR(10)     NOT NULL,
    expires_at  DATETIME        NOT NULL,
    used        TINYINT(1)      NOT NULL DEFAULT 0,
    created_at  DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (id),
    INDEX idx_phone_code (phone, code),
    INDEX idx_expires (expires_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- -----------------------------------------------------------
-- 3. 乡村表
-- -----------------------------------------------------------
CREATE TABLE villages (
    id          BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
    name        VARCHAR(100)    NOT NULL,
    description TEXT,
    cover_image VARCHAR(500),
    lat         DECIMAL(10,7)   NOT NULL,
    lng         DECIMAL(10,7)   NOT NULL,
    address     VARCHAR(300),
    tags        JSON,
    created_at  DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at  DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    PRIMARY KEY (id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- -----------------------------------------------------------
-- 4. 打卡点表
-- -----------------------------------------------------------
CREATE TABLE village_spots (
    id          BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
    village_id  BIGINT UNSIGNED NOT NULL,
    name        VARCHAR(100)    NOT NULL,
    lat         DECIMAL(10,7)   NOT NULL,
    lng         DECIMAL(10,7)   NOT NULL,
    description TEXT,
    images      JSON,
    sort_order  INT UNSIGNED    NOT NULL DEFAULT 0,
    PRIMARY KEY (id),
    INDEX idx_village (village_id),
    CONSTRAINT fk_spot_village FOREIGN KEY (village_id) REFERENCES villages(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- -----------------------------------------------------------
-- 5. 文化条目表
-- -----------------------------------------------------------
CREATE TABLE village_cultures (
    id          BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
    village_id  BIGINT UNSIGNED NOT NULL,
    type        ENUM('history','intangible','legend') NOT NULL,
    title       VARCHAR(200)    NOT NULL,
    content     TEXT,
    PRIMARY KEY (id),
    INDEX idx_village_type (village_id, type),
    CONSTRAINT fk_culture_village FOREIGN KEY (village_id) REFERENCES villages(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- -----------------------------------------------------------
-- 6. 剧本表
-- -----------------------------------------------------------
CREATE TABLE scripts (
    id                  BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
    village_id          BIGINT UNSIGNED NOT NULL,
```

## 八、设计说明

1. **JSON 灵活性**：剧本节点配置（config）、道具列表（items）等使用 JSON 字段，避免过度拆表，方便剧本编辑器灵活扩展。

2. **剧本与进度分离**：`scripts` + 子表存储剧本模板（只读），`script_progresses` 存储每个游客的游玩状态。同一份剧本被 1000 人领取，只有 1000 行 progress 记录。

3. **对话日志按需加载**：NPC 对话历史存储在 `chat_logs`，查询时按 `progress_id + npc_id` 分页拉取最近 N 条（如最近 20 轮），组装进 AI 请求上下文，不一次性加载全部。

4. **评分去重**：`ratings` 表通过 `uk_user_script` 保证一人一评。

5. **级联删除**：剧本章节/节点/任务等子表设置 `ON DELETE CASCADE`，删除剧本时自动清理子数据。

6. **ON DELETE SET NULL**：节点删除时，关联的 progress 记录 `current_node_id` 置 NULL（而非级联删除游客进度）。
