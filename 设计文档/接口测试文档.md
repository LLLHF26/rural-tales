# 接口测试文档

> 沉浸式乡村文旅剧本创作与智能导览系统 · FastAPI 后端  
> 统一响应格式：`{code: 0, message: "success", data: ...}`  
> `code=0` 表示成功，非零为业务错误码

---

## 一、移动端接口（`/v1`）

## 1A. 用户认证 → `/v1/user`

### 1A-1 POST `/v1/user/send-code` — 发送验证码

| 项 | 内容 |
|---|---|
| 认证 | 无 |
| Content-Type | `application/json` |
| Body | `{phone: string}` |

**成功响应**（code=0）
```json
{"code":0, "message":"success", "data":{"expireSeconds":300}}
```

**测试用例**

| 编号 | 场景 | 请求 | 预期 |
|------|------|------|------|
| S01 | 正常发送 | `{phone:"13800138000"}` | code=0, expireSeconds=300 |
| S02 | 开发模式 | `{phone:"13800138000"}` | 固定发送 000000（Redis key=`sms_code:13800138000`） |
| S03 | 同一号码30秒内重复发送 | 连续两次相同 phone | 第一次成功，业务层可加频率限制 |
| S04 | 无效手机号格式 | `{phone:"abc"}` | Pydantic 422 或业务层校验 |

### 1A-2 POST `/v1/user/login` — 验证码登录/注册

| 项 | 内容 |
|---|---|
| 认证 | 无 |
| Body | `{phone: string, code: string}` |

**成功响应**（code=0）
```json
{
  "code":0, "message":"success",
  "data":{
    "token":"jwt...", "expireAt":null,
    "user":{"userId":"1","nickname":"游客8000","avatar":"","phone":"138****8000","isNewUser":true}
  }
}
```

**错误码**：1001=验证码错误或已过期

**测试用例**

| 编号 | 场景 | 请求 | 预期 |
|------|------|------|------|
| L01 | 新用户首次登录 | `{phone:"13800138000", code:"000000"}` | code=0, isNewUser=true, 自动创建用户 |
| L02 | 老用户登录 | `{phone:"13800138000", code:"000000"}` | code=0, isNewUser=false |
| L03 | 验证码错误 | `{phone:"13800138000", code:"111111"}` | code=1001 |
| L04 | 验证码过期 | 等待5分钟后使用相同code | code=1001 |
| L05 | 验证码已使用 | 同一code重复登录两次 | 第二次 → code=1001 |
| L06 | 缺少phone字段 | `{code:"000000"}` | 422 |
| L07 | 空phone | `{phone:"", code:"000000"}` | 业务层校验或422 |

### 1C-2 POST `/v1/play/{progress_id}/chat` — NPC对话（SSE流式）

| 项 | 内容 |
|---|---|
| 认证 | Bearer Token |
| Query | `npcId=int, message=string, nodeId=int` |
| Response | `text/event-stream` |

**SSE事件类型**

| event | data | 说明 |
|-------|------|------|
| `message` | `{text:"..."}` | AI回复片段 |
| `done` | `{newClues:[]}` | 对话结束 |
| `error` | `{message:"..."}` | 异常 |

**测试用例**

| 编号 | 场景 | 预期 |
|------|------|------|
| CH01 | 正常对话 | SSE流返回message事件…text拼接,最后done |
| CH02 | 空消息 | `message=""` | AI应仍然回应 |
| CH03 | NPC不存在 | event=error, message="NPC不存在" |
| CH04 | 进度不存在 | event=error, message="进度不存在" |
| CH05 | 对话历史保存 | 调用后 DB中ChatLog新增两条(role=user, role=npc) |
| CH06 | SystemPrompt组装 | 含NPC设定+场景上下文+已解锁线索+NPC关系 |
| CH07 | 非思考模式 | 请求DeepSeek时 extra_body={"thinking":{"type":"disabled"}} |
| CH08 | AI服务超时 | event=error, 返回兜底文案 |

### 1C-4 POST `/v1/play/{progress_id}/task/submit` — 提交任务

| 项 | 内容 |
|---|---|
| 认证 | Bearer Token |
| Body | `{taskId:int, nodeId:int, answer?:string, photoUrl?:string, gpsLat?:float, gpsLng?:float}` |

**成功响应（成功）**
```json
{"code":0,"data":{"success":true,"message":"答案正确！","reward":{"item":{...}},"nextNodeId":"15"}}
```

**成功响应（失败）**
```json
{"code":0,"data":{"success":false,"message":"答案不太对，再想想看……","retryHint":"试试植物"}}
```

**错误码**：1001=缺少参数/无效选择, 2001=进度/任务不存在

**测试用例**

| 编号 | 类型 | 场景 | 预期 |
|------|------|------|------|
| TS01 | puzzle | 答案完全匹配(忽略大小写) | success=true, 发奖励 |
| TS02 | puzzle | 答案带空格 trim后匹配 | success=true |
| TS03 | puzzle | 答案错误 | success=false, 返回retryHint |
| TS04 | puzzle | 未传answer | code=1001 |
| TS05 | gps | 在范围内(距离≤radius) | success=true, message含"已到达" |
| TS06 | gps | 超出范围 | success=false, message含"还有约X米" |
| TS07 | gps | 未传GPS坐标 | code=1001 |
| TS08 | gps | 精确匹配(0米) | success=true |
| TS09 | choice | 任意提交 | success=true |
| TS10 | 完成任务后 | completed_task_ids增加, items增加(如有奖励), 推进到nextNodes[0] |
| TS11 | 无奖励任务 | reward=null |

**GPS距离校验说明**：使用Haversine公式计算,地球半径6371km
